Datapath-simulator

Utforska CPU:ns dataväg genom interaktiva mikro-operationer. Definiera egna styrsekvenser och se hur data flödar genom processorn.

Instruktioner:
  • Välj en fördefinierad sekvens eller skapa egen med mikro-operationer
  • Använd "Step" för att köra en operation i taget, eller "Run" för automatisk körning
  • Justera klockfrekvensen med reglaget
  • Övervaka registerinnehåll och execution trace i realtid
  • Använd Import/Export för att spara och dela konfigurationer

Mikro-operationer

Grundläggande operationer

Minnesoperationer

  • MAR ← PC - Ladda minnesadress från PC
  • MDR ← Mem[MAR] - Läs från minne
  • Mem[MAR] ← MDR - Skriv till minne
  • IR ← MDR - Ladda instruktionsregister

ALU-operationer

  • A ← R[n] - Ladda ALU-ingång A
  • B ← R[n] - Ladda ALU-ingång B
  • ALU ← A + B - Addition
  • ALU ← A - B - Subtraktion
  • ALU ← A & B - Logisk AND
  • R[n] ← ALU - Spara ALU-resultat
  • FLAGS ← ALU.flags - Uppdatera flaggor

Fördefinierade sekvenser

Fetch-Decode-Execute

Grundläggande instruktionshämtning:

  1. MAR ← PC
  2. MDR ← Mem[MAR]
  3. IR ← MDR
  4. PC ← PC + 1

Load/Store

Minnesoperationer:

  1. MAR ← R[1]
  2. MDR ← Mem[MAR]
  3. R[0] ← MDR
  4. MAR ← R[2]
  5. MDR ← R[0]
  6. Mem[MAR] ← MDR

ALU Operations

Aritmetiska operationer:

  1. A ← R[0]
  2. B ← R[1]
  3. ALU ← A + B
  4. R[2] ← ALU
  5. FLAGS ← ALU.flags

Fördjupning

Datavägens komponenter

PC (Program Counter)
Innehåller adressen till nästa instruktion som ska hämtas.
IR (Instruction Register)
Innehåller den aktuella instruktionen som ska utföras.
MAR (Memory Address Register)
Innehåller minnesadressen för läs- eller skrivoperationer.
MDR (Memory Data Register)
Innehåller data som läses från eller skrivs till minnet.
Register File
Samling av allmänna register (R[0] till R[7]) för datalagring.
ALU (Arithmetic Logic Unit)
Utför aritmetiska och logiska operationer på data.
FLAGS
Statusflaggor (Z=Zero, N=Negative, C=Carry, V=Overflow) från ALU-operationer.

Micro-operations och styrsekvenser

Varje komplex instruktion bryts ner i enkla mikro-operationer som kan utföras av hårdvaran. Styrenhet genererar dessa sekvenser baserat på instruktionstyp.

Exempel - Load-instruktion:
För att ladda data från minne till register krävs flera mikro-operationer:
  1. Sätt minnesadress (MAR ← adress)
  2. Läs från minne (MDR ← Mem[MAR])
  3. Överför till register (R[x] ← MDR)
